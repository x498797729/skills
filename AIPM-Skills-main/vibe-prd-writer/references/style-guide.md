# 前端风格方案生成规范

本文档定义了Step 4"前端风格定调"的详细执行规范，包括如何采集偏好、如何生成4个方案、如何产出Design Token。

---

## 1. 风格偏好采集

### 必问问题

按以下顺序提问（可根据用户已给出的信息跳过）：

**Q1: 产品调性**
> "你希望这个产品给用户什么感觉？可以说几个关键词，也可以说一个你喜欢的产品/网站作为参考。"

常见调性关键词供参考（不要直接列给用户，用于理解用户意图）：
- 温暖/疗愈/安全感
- 专业/高效/克制
- 年轻/活泼/有趣
- 高级/奢华/精致
- 极简/干净/呼吸感
- 科技感/未来/赛博
- 文艺/质感/手工感

**Q2: 明暗偏好**
> "你偏好亮色模式、暗色模式、还是都支持？"

**Q3: 排除项**
> "有没有你绝对不想要的风格？比如'不要太像聊天工具'、'不要紫色'之类的。"

---

## 2. 四套方案生成规则

### 差异化原则
4个方案之间必须有明显的风格差异，不能是"同一种风格的4个变体"。差异化维度包括：
- 色彩取向不同（暖色系 vs 冷色系 vs 中性色 vs 撞色）
- 排版风格不同（紧凑 vs 留白 vs 杂志式 vs 卡片式）
- 视觉重心不同（文字驱动 vs 图形驱动 vs 空间驱动）
- 情绪调性不同（内敛 vs 表达 vs 中性 vs 温暖）

### 每个方案必含内容

```markdown
### 方案[A/B/C/D]: [风格名称]

**设计理念**: [一句话，有画面感]

**氛围关键词**: [词1] · [词2] · [词3]

**色彩方案**:
- 主色: [hex] [色名]
- 辅色: [hex] [色名]  
- 强调色: [hex] [色名]
- 背景: [hex]
- 文字: [hex]
- 次要文字: [hex]

**字体搭配**:
- 标题: [字体名] ([Google Fonts链接])
- 正文: [字体名] ([Google Fonts链接])

**圆角风格**: [具体数值，如 8px / 16px / 24px / full]
**间距密度**: [紧凑 / 适中 / 宽松]
**阴影风格**: [无阴影 / 轻微 / 明显 / 多层]

**适合的产品类型**: [什么类型的产品用这个风格最合适]
```

### 方案命名规范
名称要有画面感和辨识度，例如：
- ✅ "北欧晨雾" "深海静谧" "暮光花园" "月球基地"
- ❌ "方案1" "蓝色风格" "简约版" "深色版"

## 3. 可视化预览生成

### 预览内容要求
为4个方案各生成一个可视化预览，展示以下元素：
- 背景色 + 主色/辅色/强调色色块
- 标题字体 + 正文字体预览文字
- 一个按钮样式（主要 + 次要）
- 一个卡片样式
- 整体氛围展示

### 预览生成方式
使用Visualizer工具（show_widget）生成一个包含4个方案并排展示的HTML预览。

预览要求：
- 4个方案水平或网格排列，方便对比
- 每个方案展示完整的视觉元素
- 方案标题和关键词清晰可见
- 用户可以直观感受色彩、字体和整体氛围的差异

### 字体引入
在预览HTML中引入Google Fonts：
```html
<link href="https://fonts.googleapis.com/css2?family=[Font1]&family=[Font2]&display=swap" rel="stylesheet">
```

注意选择支持中文的字体或提供中文fallback方案：
- 中文标题推荐：Noto Serif SC, ZCOOL XiaoWei, Ma Shan Zheng, ZCOOL QingKe HuangYou
- 中文正文推荐：Noto Sans SC, LXGW WenKai
- 也可以用系统中文字体作为fallback：`-apple-system, "PingFang SC", "Hiragino Sans GB", "Microsoft YaHei", sans-serif`

## 4. 用户选择与微调

### 选择流程
1. 展示4个可视化预览
2. 请用户选择一个偏好方案（或混搭："喜欢A的配色+B的字体"）
3. 如果用户要微调，执行微调后重新展示确认
4. 确认后锁定设计语言

### 微调支持项
用户可以微调以下内容：
- 更换某个色值
- 更换字体
- 调整圆角大小
- 调整间距密度
- 增加/减少暗色模式支持

### 锁定后的产出
确认的方案将直接转化为模块3（Design Tokens）的CSS变量定义。

## 5. 风格库（预设参考）

以下是一些高质量的风格参考方向，帮助生成灵感。不要照搬，而是基于用户偏好进行创造性组合。

### 疗愈/温暖系
- 大量使用暖白/米色/淡粉/鼠尾草绿
- 圆润形态（大圆角16-24px）
- 宽松间距，大量留白
- 字体偏柔和的衬线体或圆体
- 参考：Headspace, Calm, Things 3

### 专业/克制系
- 中性色为主（灰/深蓝/白）+ 单色强调
- 适中圆角（6-12px）
- 紧凑但不拥挤的信息密度
- Sans-serif字体，偏几何
- 参考：Linear, Notion, Stripe

### 年轻/活力系
- 明亮的强调色（荧光黄/电光蓝/珊瑚红）
- 中等圆角 + 偶尔的圆形元素
- 适中间距
- 粗体/黑体标题 + 清爽正文
- 参考：Discord, Figma, Spotify

### 高端/精致系
- 深色背景 + 金色/银色/低饱和度强调
- 小圆角（4-8px）或直角
- 宽松间距，留白大
- 优雅的衬线体标题 + 精致sans正文
- 参考：Apple, Porsche, Aesop

### 科技/未来系
- 深色背景 + 霓虹色/渐变强调
- 直角或小圆角
- 紧凑信息密度
- Mono或几何Sans字体
- 参考：Vercel, GitHub, Raycast

### 文艺/质感系
- 米白/纸张色背景 + 深棕/墨绿强调
- 中等圆角
- 宽松间距
- 衬线体为主
- 参考：Medium, Bear, Craft
