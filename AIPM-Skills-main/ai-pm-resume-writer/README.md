# AI产品经理简历撰写 Skill

> 一份优秀的简历是一个产品——目标用户是HR和面试官，核心价值是让他们在10秒内判断：这个人就是我们要找的。

这个Skill帮助AI产品经理完成从零撰写、诊断优化、JD适配的简历全流程，输出专业的HTML格式简历文件。

## 安装方式

### 在 Claude Desktop / Cowork 中使用

1. 将整个 `ai-pm-resume-writer` 文件夹放入你的 Skills 目录：
   - macOS: `~/Library/Application Support/Claude/skills/user/ai-pm-resume-writer/`
   - 或者你的自定义 Skills 路径下
2. 确保目录结构如下：
   ```
   ai-pm-resume-writer/
   ├── SKILL.md              # 核心skill指令
   ├── README.md             # 本说明文件
   └── references/           # 参考文件（skill会自动读取）
       ├── writing-guide.md
       ├── examples-by-tier.md
       └── diagnosis-checklist.md
   ```
3. 重启 Claude Desktop，skill即自动生效

### 在 Claude.ai 网页版中使用

将 `SKILL.md` 的内容粘贴到 Project 的 System Prompt 中，同时将 `references/` 下的文件作为 Project Knowledge 上传。

## 使用前准备

### 你需要准备什么素材？

这个skill的核心理念是**"交素材，而非填表格"**——你不需要自己组织简历语言，只需要把原始素材丢给它，它来帮你挖掘亮点。

**最推荐的素材（效果最好）：**

| 素材类型 | 为什么有用 | 示例 |
|---------|----------|------|
| **PRD / 产品方案文档** | 能直接提取项目背景、技术方案、你的设计决策 | 你写过的任何PRD、方案doc |
| **绩效自评 / 月报** | 藏着你自己都忘了的成果和数据 | 绩效填写的Excel、月度总结 |
| **目标岗位的JD** | 让简历精准命中HR的需求 | 招聘网站截图或链接 |

**也可以用这些（有比没有好）：**

| 素材类型 | 说明 |
|---------|------|
| 旧简历（哪怕很粗糙） | 作为骨架来优化 |
| 产品截图 / 链接 | 反向推导你做过的工作 |
| 竞品分析文档 | 展示你的分析深度 |
| 日报 / 周报 | 识别关键动作 |

**什么都没有也行：**

直接用大白话聊你最近在忙什么、做过什么觉得有成就感的事，skill会通过追问帮你挖掘。

### 你需要知道的信息

开始对话时，skill会依次问你：

1. **你现在做什么工作？做了多久？**（判断经验层级）
2. **目标薪资范围**（15-20k / 20-30k / 30-40k，决定写法策略）
3. **有没有目标JD？**（如果有，简历会针对性适配）

准备好这些信息，对话会更高效。

## 三种使用模式

### 模式A：从零撰写

**适用场景**：没有简历，或者简历太旧需要重写

**触发方式**：直接说"帮我写简历"、"我要找工作了"

**流程**：
```
定方向 → 收素材 → 深度挖掘亮点 → HR视角分析 → 确认人设和排序 → 生成HTML
```

### 模式B：诊断优化

**适用场景**：已有简历，想知道哪里有问题

**触发方式**：上传简历文件 + 说"帮我看看简历"

**流程**：
```
读取简历 → 三级诊断扫描 → 输出诊断报告（🔴🟡🟢） → 确认后执行优化
```

### 模式C：JD适配

**适用场景**：有简历有目标JD，想针对性调整

**触发方式**：上传简历 + 贴JD + 说"针对这个岗位调简历"

**流程**：
```
拆解JD → 匹配度评估 → 输出调整建议 → 确认后执行适配
```

三种模式可以叠加，比如"上传简历 + 贴JD"就是B+C组合。

## 对话示例

### 最简单的开始方式

```
你：帮我写一份AI产品经理的简历

Skill：根据你提供的信息，我建议用从零撰写模式来推进。先聊两个基础问题：
      1. 你现在做什么工作？做了多久了？
      2. 有没有已经看好的目标岗位或JD？
```

### 带素材的开始方式

```
你：帮我写简历，我上传了一份PRD文档作为参考 [上传文件]

Skill：我从你的文档里挖出了这些可以写进简历的亮点：
      1. Multi-Agent架构设计能力...
      2. 三端角色深度拆解...
      这些准确吗？另外我想追问几个细节...
```

### 诊断模式

```
你：帮我看看这个简历有什么问题 [上传简历]

Skill：简历诊断报告
      总体评价：框架完整但缺乏AI特色和量化数据
      🔴 必须修复（3项）...
      🟡 建议修复（2项）...
      🟢 做得好的地方...
```

## 拿到HTML文件后怎么用

Skill输出的是一个 `.html` 文件，这是最灵活的格式——以下是几种常见的后续操作。

### 方式一：直接打印为PDF（最快）

1. 在浏览器中打开HTML文件（双击即可）
2. 按 `Ctrl+P`（Mac: `Cmd+P`）打开打印对话框
3. 目标打印机选择 **"另存为PDF"**
4. 页边距选择 **"无"** 或 **"最小"**
5. 勾选 **"背景图形"**（如果有的话）
6. 点击保存

> 💡 打印时，蓝色的"建议补充"标记会自动隐藏，只保留正式内容。

### 方式二：导入Figma转设计稿

**方法A：使用html.to.design插件（推荐）**

1. 在Figma中安装插件 [html.to.design](https://www.figma.com/community/plugin/1159123024924461424)
2. 在浏览器中打开你的HTML简历文件
3. 复制浏览器地址栏中的地址（如果是本地文件，地址会是 `file:///...`）
4. 在Figma中运行插件 → 粘贴地址或直接粘贴HTML代码
5. 插件会将HTML转换为Figma的可编辑图层
6. 之后你可以在Figma中自由调整字体、颜色、间距、排版

**方法B：截图后导入Figma**

1. 在浏览器中打开HTML文件
2. 使用浏览器全页截图（Chrome: `Ctrl+Shift+I` → `Ctrl+Shift+P` → 输入"screenshot" → 选"Capture full size screenshot"）
3. 将截图拖入Figma作为底图
4. 在底图上方重新绘制文本框和排版

**方法C：先转PDF再导入Figma**

1. 用方式一生成PDF
2. 在Figma中安装 [PDF to Figma](https://www.figma.com/community/plugin/1231439613393985567) 插件
3. 导入PDF，插件会将其转换为可编辑的Figma元素

### 方式三：转为可编辑的Word/Google Docs

**方法A：通过Google Docs**

1. 打开 [Google Docs](https://docs.google.com)
2. 文件 → 打开 → 上传 → 选择你的HTML文件
3. Google Docs会自动将HTML转换为可编辑文档
4. 可以继续导出为 `.docx` 格式

**方法B：通过WPS/Word直接打开**

1. 右键HTML文件 → 打开方式 → 选择 Microsoft Word 或 WPS
2. Word会将HTML渲染为文档格式
3. 另存为 `.docx`

> ⚠️ 注意：HTML转Word后排版可能会有偏差，建议做微调。如果对排版要求高，推荐走Figma路线。

**方法C：用pandoc命令行转换**

```bash
pandoc resume.html -o resume.docx
```

### 方式四：直接编辑HTML源码

如果你懂一点HTML/CSS，可以直接用任何文本编辑器（VS Code、Sublime Text等）打开HTML文件修改内容：

- 所有文本内容都是纯中文，搜索替换即可
- `<span class="ph">待确认：XX</span>` → 搜索 `ph` 替换为你的真实数据
- `<div class="suggest">...</div>` → 删除这些蓝色建议块（或保留作为备忘）

## 输出文件说明

HTML文件中有两种特殊标记：

| 标记 | 颜色 | 含义 | 你需要做什么 |
|------|------|------|-------------|
| `待确认：XX` | 🟡 黄色底 | 需要你填入真实数据的占位符 | 替换为真实数据，面试时每个数据都可能被追问 |
| `💡 建议补充：...` | 🔵 蓝色底 | Skill建议你补充但无法代写的内容 | 考虑是否补充，打印PDF时自动隐藏 |

## 常见问题

### Q：我完全没有AI产品经验，能用这个skill吗？

可以。skill支持"15-20k初级/转型"层级，会重点挖掘你在原岗位中做过的与AI PM相关的工作（如参与需求讨论、做过用户调研、自学过Prompt Engineering等），以及Side Project。

### Q：我不知道具体数据怎么办？

skill会用`待确认：XX`标记所有需要数据的位置，你可以之后自己查询系统或回忆估算后替换。skill不会帮你编造数据——每个数据面试时都可能被追问"怎么算的"。

### Q：为什么不输出docx格式？

简历排版对细节要求很高，docx在处理复杂布局（尤其是双栏、文本框嵌套）时容易出现兼容性问题。HTML是最稳定的格式，可以精确控制每个像素，而且转PDF/Figma/Word都很方便。

### Q：HTML打印出来分页位置不对怎么办？

在浏览器打印设置中：
- 缩放比例调为 100%
- 页边距选"无"或"自定义"设为 0
- 如果内容超出一页，可以回到HTML中适当精简内容，或调整CSS中 `.page` 的 `padding` 值

### Q：我想要不同的视觉风格怎么办？

当前HTML模板是纯黑白灰的简约风格，这是AI产品经理简历最安全、最专业的选择。如果你确实需要调整：
- **推荐方式**：导入Figma后在Figma中调整颜色、字体、间距
- **进阶方式**：直接编辑HTML文件中的CSS变量（`:root` 下的颜色值）

### Q：skill不会自动帮我完善简历细节吗？

skill的理念是**教学导向**——它会挖掘你的亮点、给出写法建议、解释为什么这样写更好，但不会替你编造经历或虚构数据。它输出的内容可能比你提供的多（会建议你补充方向），但所有标了`待确认`和`建议补充`的地方都需要你自己完善。

### Q：如何获得最好的效果？

1. **多给素材**：PRD、周报、绩效自评越多越好，skill从中挖掘的亮点比你自己想的多
2. **认真回答追问**：skill的追问都是有目的的，尤其是"踩坑故事"和"决策逻辑"，这些是让你简历独特的关键
3. **提供目标JD**：有JD的情况下，简历的针对性会大幅提升
4. **拿到初稿后多迭代**：第一版出来后，可以继续对话要求修改特定部分
