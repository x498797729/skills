## 经验层级判断规则

| 层级 | 薪资段 | 典型特征 |
|------|--------|----------|
| 初级/转型 | 15-20k | 0-1年经验、实习转正、非PM岗转型（UI、客服、运营、测试）、应届生 |
| 成长期 | 20-30k | 1-3年经验、已有独立负责功能模块的经验、开始独立负责完整项目 |
| 资深 | 30-40k | 3-5年经验、独立负责过完整产品线、有商业化/团队管理经验 |

**核心区别：**
- 15-20k 证明"我能做" → 侧重执行力、学习能力、AI实践动手能力
- 20-30k 证明"我能独立做好" → 侧重需求分析深度、数据驱动迭代、AI技术理解深度
- 30-40k 证明"我能带着做大" → 侧重产品战略规划、商业化落地、团队赋能

## 项目经历示例

### 15-20k：个人实践项目（Side Project）

```
小红书文案生成Agent | 产品设计者 & 开发者 | 2024.06-2024.08

【S-背景】
观察到身边运营同事每天花大量时间写小红书文案，平均单篇花费30-60分钟，且效果波动大。
市面上的文案生成工具多为通用型，缺乏对小红书平台特性的深度理解。

【T-目标】
基于Coze平台，搭建一个专注小红书场景的文案生成Agent，目标让运营人员的文案创作效率
提升至少5分钟/篇，且生成内容采纳率>30%。

【A-行动】
• 用户调研：访谈8位运营同事，整理出3个核心痛点：①风格不稳定、②标题点击率低、
  ③缺少hook和互动引导。同时收集各品类下50篇爆款文案作为参考样本。
• 知识库搭建：将50篇爆款文案按品类、风格、结构进行标注分类，构建小红书风格知识库，
  纳入平台规则、标题公式、爆款话术模板等结构化数据。
• Prompt设计：设计三步走工作流：①用户输入产品信息与目标人群→②Agent检索知识库
  匹配对应品类风格→③生成包含标题+正文+标签的完整文案。采用Few-shot示例引导风格一致性。
• 测试与迭代：邀请8位运营同事试用，收集反馈并进行了3轮优化，重点解决了"生成内容
  太僵硬/广告感强"的问题，最终确定了"发现型+故事型"两套核心模板。

【R-结果】
• 生成文案的运营采纳率达到32%（直接使用或小幅修改后使用），单篇创作时间从平均>30
  分钟缩短至5分钟。
• Agent在朋友圈小范围传播，累计使用用户40+人，获得"很懂小红书的调性"等正面反馈。
• 形成了一套可复用的「小红书场景化Prompt写作方法论」，沉淀为个人文章发布在[人人都是
  产品经理]。
```

| 层级 | S(背景) | T(目标) | A(行动) | R(结果) | L(学习) |
|------|---------|---------|---------|---------|---------|
| 15-20k | 1-2行 | 1行 | ★重点展开3-5点 | 2-3点 | 可省略 |
| 20-30k | 2-3行 | 1-2行 | ★重点展开5-8点 | ★3-4点 | 可选 |

**15-20k的A(行动)重点：** 展示你做了什么具体工作、用了什么方法
**20-30k的A(行动)重点：** 展示你的思考过程——为什么这么做、对比了什么方案
**30-40k的A(行动)重点：** 展示你的方法论和架构能力——如何设计体系、如何赋能团队

## 中高级核心原则：指标思维与体系化思维

### 什么是指标思维

中高级PM的简历，每一条经历都应该呈现一条完整的指标链路：

```
发现指标问题 → 分析根因 → 采取行动/做出决策 → 提升了什么指标
```

所有子指标必须服务于顶级业务指标，形成一棵指标树。面试官看完每一条都应该想追问"这个数据怎么算的？为什么选这个方案？"——这才是好简历。

### 指标树示例

**ToC增长场景的指标树：**
```
顶级指标：产品ARR从150万→850万
├── 付费转化率提升127%
│   └── 动作：50+次用户访谈+A/B测试重构付费路径
├── 用户7日留存率从32%→58%
│   └── 动作：AARRR模型+新手引导流程设计
├── DAU从1.2万→8.7万
│   └── 动作：KOL合作+社区运营+推荐算法优化
└── 内容点击率提升43%
    └── 动作：LLM+向量数据库构建用户画像，千人千面分发
```

**ToB交付场景的指标树：**
```
顶级指标：累计签约金额1200万+，复购率65%
├── 新项目交付周期从45天→20天
│   └── 动作：设计四层产品架构+标准化AI中台能力
├── 客户满意度NPS达78分
│   └── 动作：场景化需求拆解+POC快速验证
├── POC线索转化率从8%→21%
│   └── 动作：与售前共建解决方案话术库与Demo库
└── 模型迭代效率提升35%
    └── 动作：建立包含12项核心指标的模型评测体系
```

**RAG产品场景的指标树：**
```
顶级指标：产品日均查询120+次，问答准确率84%
├── 用户采纳率从58%→78%
│   └── 根因：知识库覆盖率仅65% → 扩充至800+篇+优化分块策略
├── 评测综合得分从68→84分
│   └── 动作：设计三维评估体系+标注200条评测集+3轮Prompt迭代
└── 新员工上手时间从2周→5天
    └── 动作：设计「搜索+问答+参考源展示」三层交互架构
```

### 写法公式

每条bullet = [发现XX指标存在XX问题] + [通过XX方法/做了XX决策] + [使XX指标从A提升至B]

### 正反对比

**❌ 没有指标思维（堆砌动作）：**
> 负责RAG知识问答模块的产品设计，完成了PRD撰写、原型设计、知识库搭建、Prompt优化、模型评估等工作，产品成功上线。

**✅ 有指标思维（指标驱动叙事）：**
> 负责RAG知识问答模块，上线后发现采纳率仅58%。通过数据分析定位根因为知识库覆盖率不足（仅覆盖65%高频问题），主导知识库从400篇扩充至800+篇并优化分块策略，同步进行3轮Prompt迭代（评测集综合得分从68→84分），最终将采纳率从58%提升至78%，日均查询量稳定在120+次。

**❌ 没有体系化思维（零散罗列）：**
> • 搭建了数据看板
> • 做了A/B测试
> • 建立了评测数据集
> • 优化了Prompt
> • 培训了业务团队

**✅ 有体系化思维（体系驱动）：**
> 搭建产品数据监控体系（覆盖准确率、采纳率、响应时间等8项核心指标），建立"数据异常→根因分析→A/B验证→上线迭代"的标准化优化闭环。通过该体系发现并解决6个关键瓶颈：如知识库覆盖率不足导致采纳率低（58%→78%）、Prompt模板僵化导致金融场景得分低（68→84分）等，整体转化漏斗效率提升38%。

### 如何帮用户建立指标思维

在A1信息收集阶段，当用户描述经历时，主动用以下追问帮他们梳理指标链路：

1. **找顶级指标**："你们这个产品/项目最核心的业务指标是什么？老板最关心什么数字？"
2. **找子指标**："这个指标受哪些因素影响？你负责的部分影响的是哪个子指标？"
3. **找指标问题**："你接手的时候，这个指标是多少？有什么问题？"
4. **找决策逻辑**："你当时有几个方案可选？为什么选了这个？"
5. **找结果**："做完之后指标变成了多少？怎么算出来的？"

这五个问题问完，一条有指标思维的bullet就能写出来了。
