# AI产品经理简历撰写方法论

## 核心理念

一份优秀的简历是一个产品，不是个人经历的流水账。
- 目标用户：HR和业务面试官
- 核心价值：让他们知道——我，就是他们要找的能解决问题的人
- 三大原则：成果导向、重点突出、量化思维

---

## 准备阶段：四步法

### 第一步：拆解JD
1. 逐字逐句阅读JD
2. 提炼核心关键词、硬性技能要求（如RAG、数据分析）和软性实力要求（如跨团队协作）
3. 用一句话总结：这家公司现在最需要一个能解决什么核心问题的人？

**输出格式：**
- 核心问题：一句话
- 核心关键词：列出5-8个
- 硬性技能：列出3-5个
- 软性实力：列出2-3个

### 第二步：盘点个人能力
罗列从大学至今所有项目经历、工作职责、实习内容、个人作品、奖项、技能。
每个项目/职责下写出：职责 → 行动 → 成果（附具体数字）。

### 第三步：匹配
- 拿JD需求清单和个人能力素材库做匹配
- 二八原则：找出20%最能回应岗位核心需求的亮点，作为简历主角
- 亮点先行：最亮眼的经历排最前面
- 与岗位关联度不高的经历，果断舍弃

### 第四步：构建叙事框架
- 个人优势用什么开场白在5秒内抓住眼球？
- 哪些项目详细展开，哪些一笔带过？
- 所有模块串联起来，讲述一个逻辑自洽且高度统一的故事
- 明确你是什么样的人、如何成长、学习方法、未来目标与规划

---

## 简历模块撰写规则

### 一、个人信息

**必写：** 姓名 | 电话 | 邮箱 | 求职意向：AI产品经理｜工作状态
**可写：** LinkedIn | 作品集/人人都是产品经理/小红书/博客/GitHub链接
**不写：** 政治面貌 | 婚姻状况 | 身高体重

注意：邮箱用专业名称，避免随意昵称。

### 二、个人优势

必须在3-5行内抓住眼球。

**公式：** 一句话优势点 = [经验/能力] + [关键动作/方法论] + [亮眼成果/价值]

**规则：**
- 关键词驱动：提炼核心能力标签作为小标题
- 亮点前置：最亮眼的经验和数字放最前面
- AI相关性：包含热门词汇（让HR懂）和冷门词汇（让技术面试官认可）
- 结果导向：用一个最亮眼的可量化成绩来证明能力
- **禁忌：千万不要写"认真负责、勤奋努力、上进心强"，只会减分**

### 三、核心技能（可选模块）

**规则：**
- 分类清晰：产品能力 / AI技术理解 / 数据分析 / 跨职能协作 / 工具
- 具体化：不要只写"AI"，要写大模型应用、RAG、Agent、CE、PE等
- 与项目匹配：技能点能在项目经验中找到印证
- **禁忌：不要只列工具名称**

### 四、工作经历

**公式：** 成就 = [动词] + [工作内容/项目] + [方法] + [量化的业绩/成果]

**规则：**
- 由近到远排列
- 核心成就用项目符号列出3-5条，而非岗位描述罗列
- 写事实而非职责：不写"负责产品规划"，写"推动了XX功能上线，带来XX%付费转化提升"
- 聚焦个人贡献：在团队项目中清晰说明"我的部分"
- 数据可追溯：每个数据都要经得起提问

**工作经历与项目经历的区别：**
- 工作经历 = 在一家公司的整体工作概述（可能包含多个项目+协助工作）
- 项目经历 = 挑选最有代表性的项目深入展开

### 五、项目经历（STARL法则）

**格式：**
```
项目名称(突出核心价值) | 你的角色 | 项目周期

【S-Situation 背景】(2-3行)
- 业务背景 + 核心问题(用数据说明严重性) + 战略意义

【T-Task 任务】(1-2行)
- 可量化的目标 + 你的职责角色

【A-Action 行动】(核心，3-8个bullet points)
按时间或重要性排序：需求调研 → 方案设计 → 开发协同 → 测试优化 → 上线运营

【R-Result 结果】(2-5个bullet points)
- 业务指标 + 用户价值 + 商业价值 + 团队影响

【L-Learning 学习】(可选，面试时口述更好)
- 方法论沉淀 + 踩过的坑
```

**Action中必须体现AI PM的独特工作：**
定义数据标注规则、参与特征工程讨论、设计A/B测试验证模型效果、Prompt设计与优化、模型评估与选型等。

### 六、教育背景

- 公式：学校名称 | 专业 | 学历 | 在校时间
- 如果学校不是顶尖名校，放在工作和项目经历之后
- 相关专业（计算机、数学、信息管理等）是加分项
