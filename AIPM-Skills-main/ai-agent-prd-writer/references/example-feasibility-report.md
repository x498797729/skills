# 参考案例：工具型AI产品 - 可研报告生成系统

> 本文件为PRD参考结构，生成工具型AI产品PRD时可参考其模块组织和内容深度。

## 产品概述
- **产品名称**：可研报告生成系统 V2.0
- **产品类型**：工具型AI产品（AI辅助写作工具）
- **目标用户**：氢能行业研究员、管理员
- **核心场景**：将5-6天的可研报告撰写周期压缩至≤2天

## 值得借鉴的结构特点

### 1. 背景部分用"现状→痛点→机会点"三维表格
把抽象的业务问题拆解为业务、数据、专家知识、质量风险四个维度，每个维度都有现状、痛点和机会点的具体描述。这让所有利益相关者都能快速理解为什么要做。

### 2. 指标体系分四层
- 北极星指标：采纳率≥52%（衡量AI内容对用户是否有用）
- 商业指标：营收同比提升≥15%
- 业务指标：单份报告工时≤2天
- 技术指标：事实性错误率≤18%

每个指标都有明确的度量方式和数据来源。

### 3. 用户画像用"角色卡片"形式
为"李工"这个虚拟研究员创建了完整的人物卡片：学历背景、从业年限、核心任务、痛点清单、期望描述。让团队对用户形成共同认知。

### 4. 功能清单表格含验收标准
每个功能点都有一句话的验收标准，如"点击高亮句弹出原文、文档名、页码，准确率≥95%"。开发拿到就能测试。

### 5. Agent设计包含全局上下文管理
定义了doc_config、global_facts、writing_memory、runtime_data四层上下文结构，确保多Agent协作时数据一致。

### 6. 提示词设计结构完整
每个Agent（大纲生成、内容生成、全局优化、纠错）都有完整的：Role → Inputs → Action Path → Tool Use Rules → Output Specifications → Constraints。

### 7. 兜底策略用三列表格
触发条件 → 系统行为 → 用户感知，覆盖了向量召回低分、文档解析失败、AI超时、网络中断等场景。

### 8. 非功能需求按类别组织
性能、可靠性、安全性、易用性、兼容性、可维护性六大类，每类都有具体的量化要求。

## 关键模块组成
```
基础信息（版本、术语）
→ 项目背景（需求背景、项目目标、用户画像、竞品分析）
→ 功能规划（功能清单表）
→ 功能需求（流程图、交互设计表、提示词设计、全局上下文、埋点、权限）
→ 非功能需求
→ 数据需求（评测集）
→ 兜底策略
```
