# 参考案例：Agent型AI产品 - 轮胎制造根因分析系统

> 本文件为PRD参考结构，生成Agent型产品PRD时可参考其模块组织和内容深度。

## 产品概述
- **产品名称**：智见（InsightPro）- 工业根因分析智能引擎
- **产品类型**：Agent型AI产品（数据分析+知识检索Agent）
- **目标用户**：QA/QC质检人员、工艺工程师、管理层
- **核心场景**：轮胎生产缺陷的自动化根因定位与分析报告生成

## 值得借鉴的结构特点

### 1. 用典型场景替代抽象需求描述
不是写"系统需要支持数据查询功能"，而是用完整的对话场景来展示：
- 数据调查分析场景：QC输入问题→系统解析→建议查询方向→展示数据→生成报告
- 知识库查询场景：工艺员提问→系统查询相关数据→展示分析→给出建议

### 2. Agent工作流可视化
用流程图清晰展示：
- 原始业务流程（用户现在怎么做）
- Agent工作流（系统怎么做）
- 两者的对应关系

核心Agent链路：query → query_rewrite → intention → thinking_root_cause_analysis → SQL_generation → SQL_execute → visual_display → report_generation

### 3. 提示词设计按Agent角色分离
每个Agent有独立的提示词，职责清晰：
- **意图识别Agent**：分析用户输入，分类问题类型，提取关键词
- **SQL生成Agent**：根据模板和用户输入生成查询语句
- **SQL反思Agent**：当SQL执行失败时分析错误并修正
- **数据分析Agent**：对查询结果进行业务分析
- **可视化Agent**：选择合适图表类型并生成
- **报告生成Agent**：汇总所有分析结果生成结构化报告

### 4. 输出形式多样化设计
不是只有一种输出格式，而是根据场景自适应：
- Quick Answer（1-2分钟）：现场快速诊断
- Analysis Memo（5-10分钟）：班组讨论
- Detailed Report（30分钟+）：正式汇报
- Summary（即时）：日报/周报
- Exploratory（任意）：深度挖掘

### 5. 基于原则而非固定模板的输出控制
定义了四类输出原则（而非固定格式）：
- 数据可信性原则
- 逻辑严谨原则
- 可执行性原则
- 用户友好原则

并根据数据充分度、用户技术水平、紧急程度动态调整输出深度。

### 6. 兜底话术按场景分类
- 缺少关键查询实体 → 引导用户补充
- 知识库信息缺失 → 请用户提供或上传
- Agent内部错误 → 建议重试
- 数据不足 → 说明可能原因
- 能力边界 → 诚实说明局限并提供替代方案

### 7. 完整的输出示例
提供了一份深度根因分析报告的完整示例（约500行），包含：
- 问题概述与严重程度评估
- 关键指标对标分析（当前值vs标准值vs历史平均）
- 多维度根因分析（每个维度有数据证据、工艺原理、正反证据）
- 综合根因判定（主因+次要+排除+置信度）
- 行动方案与改善计划（立即/短期/中期）
- 数据局限性声明
- 后续追踪机制

### 8. 数据需求明确
- 病疵SOP知识库：需要结构化的问题-解决方案映射
- 数据字典：明确每个字段的含义、数据范围、关联字段
- 原始数据表：生产过程的实际数据
- 评测集：根因分析测试用例

## 关键模块组成（Agent型侧重）
```
背景与典型场景
→ Agent工作流设计（原始流程 vs Agent流程）
→ LangGraph运行流程
→ 各Agent提示词设计（意图识别、SQL生成、数据分析、可视化、报告生成）
→ 输出形式多样化设计（自适应输出原则）
→ 兜底话术
→ 数据需求（知识库、数据字典、评测集）
→ 验收指标
```

## Agent型PRD与工具型的核心差异

| 维度 | 工具型PRD | Agent型PRD |
|------|---------|-----------|
| 重点章节 | 详细交互设计、页面原型 | Agent工作流、提示词设计、工具调用链 |
| 流程图 | 用户操作流程为主 | Agent决策链路为主 |
| 输出定义 | 页面展示规范 | 输出原则+多形式适配 |
| 兜底策略 | UI层面的异常提示 | Agent层面的降级和话术 |
| 数据需求 | 存储和展示 | 知识库结构、数据字典、评测集 |
| 质量标准 | 交互准确性 | 意图识别率、SQL正确率、任务成功率 |
