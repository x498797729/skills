# 引导模式问题库

当用户需求模糊时，按以下维度逐步引导。**每次只问一个问题，等用户回答后再继续下一个。**

---

## 第一维度：Why & Why Now（最先问）

**目标**：让用户讲清楚为什么要做这个东西，以及为什么现在就要做。

**提问顺序**：

1. "你想解决的核心问题是什么？能用一句话描述吗？"
   - 如果用户回答太抽象，追问："能举个具体的例子吗？比如某个用户在某个场景下遇到了什么困难？"

2. "这个问题现在有多严重？有没有具体的数据可以说明？"
   - 引导量化：如耗时多久、出错率多少、成本多高、客户投诉多少次

3. "为什么现在要解决这个问题？（而不是三个月后、半年后）"
   - 引导紧迫性：如客户催了、市场窗口期、竞品已上线、业务瓶颈加剧

4. "如果这个问题不解决，最坏的情况是什么？"
   - 帮助用户理解问题的影响范围

**完成标准**：能写出一段有说服力的需求背景（包含具体数据和紧迫性理由）。

---

## 第二维度：Who（用户画像）

**目标**：搞清楚谁会用这个产品，他们的真实工作场景是什么。

**提问顺序**：

1. "这个产品的核心用户是谁？他们的角色和专业背景是什么？"
   - 不要"所有人"，要具体的角色

2. "能描述一下他们的典型工作日吗？在什么场景下会需要用到这个功能？"
   - 引导场景化思考

3. "他们现在是怎么完成这件事的？完整的操作步骤是什么？"
   - 梳理现有流程，找出痛点

4. "在这个流程中，最让他们痛苦/浪费时间的环节是哪一步？"
   - 定位具体的痛点

**完成标准**：能写出包含角色、场景、核心任务、痛点、诉求的用户画像卡片。

---

## 第三维度：What（做什么）

**目标**：明确产品要做的核心功能，以及不做什么。

**提问顺序**：

1. "如果这个产品做成了，用户的工作流程会变成什么样？"
   - 引导用户想象理想状态

2. "为了实现这个效果，你觉得产品必须要有哪些核心功能？"
   - 列出功能清单

3. "这些功能中，哪些是第一版必须有的？哪些可以后面再做？"
   - 排优先级

4. "有没有明确不做的事情？"
   - 定义Scope，防止需求蔓延

5. "你怎么知道这个功能做成功了？有什么指标可以衡量？"
   - 定义成功标准

**完成标准**：能写出功能清单（含优先级）和北极星指标。

---

## 第四维度：How（怎么做）—— Agent/AI 相关

**目标**：梳理AI实现方案的关键问题。仅在产品涉及AI时使用。

**提问顺序**：

1. "这个功能在用户的操作流程中，AI在哪些环节介入？"
   - 定位AI的作用点

2. "AI的输入是什么？（用户提供什么、系统提供什么）"

3. "AI的输出是什么？输出给谁看？输出格式是什么？"

4. "AI处理过程中需要调用哪些数据源或工具？"
   - 如：知识库检索、数据库查询、外部API

5. "对AI输出的质量要求是什么？准确率需要达到多少？"
   - 量化质量标准

6. "如果AI给出了错误或不确定的结果，应该怎么处理？"
   - 引导兜底策略

7. "用户如何反馈AI的结果好不好？有没有纠错机制？"
   - 引导数据闭环

**完成标准**：能画出系统处理流程图，能定义每个Agent的输入输出。

---

## 第五维度：数据与基础设施

**目标**：评估数据和技术基础是否就绪。

**提问顺序**：

1. "这个功能需要用到哪些数据？这些数据现在在哪里？"

2. "数据是干净的还是需要清洗？有没有缺失值、格式不统一的问题？"

3. "团队现在用什么技术栈？有没有已经在用的AI服务或模型？"

4. "有没有现成的测试数据集可以验证效果？"

**判断分支**：
- 数据就绪 → 正常生成PRD
- 数据需要治理 → PRD中增加数据清洗模块
- 数据完全缺失 → 建议先做数据准备，PRD中标注前置依赖

---

## 快速完备性检查

在进入PRD生成之前，对照检查是否已收集到足够信息：

### 必须有（缺任何一项都不开始生成）
- [ ] 要解决的核心问题是什么 + 为什么现在要做
- [ ] 核心用户是谁 + 他们的痛点
- [ ] 核心功能清单 + 优先级
- [ ] 成功标准（至少一个可量化指标）

### 应该有（缺失时标注跳过）
- [ ] 用户现有工作流程
- [ ] 竞品参考
- [ ] 数据现状
- [ ] 技术约束
- [ ] 非功能需求

### 可以没有（后续补充）
- [ ] 详细的交互设计
- [ ] 完整的提示词/Agent设计
- [ ] 埋点方案
- [ ] 角色权限
